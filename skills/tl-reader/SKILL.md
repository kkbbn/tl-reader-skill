# tl-reader

## 目的

ブルーアーカイブの戦闘動画から、有料コストで実行されたEXスキルのタイムラインを作る。出力は次の形式にする。

```text
7.7 (3:43.500) カヨコ
3.1 (3:42.633) クルミ
```

初版では0コスト化されたEXスキルは無視する。カード消失だけで判断せず、必ずコスト減少も確認する。

## 呼び出し方

- Claude Code: `/tl-reader <URLまたはMP4パス>`
- Codex: `$tl-reader <URLまたはMP4パス>` または `/skills` から選択する

Codex CLI の `/...` は主に組み込みコマンド用で、リポジトリ共有のskillは `$tl-reader` として明示呼び出しする。

## 実行手順

1. リポジトリルートで準備スクリプトを実行する。

```bash
python3 skills/tl-reader/scripts/prepare_timeline.py "<URLまたはMP4パス>"
```

2. スクリプトが出力した `review.md` を読む。MP4が `~/Downloads/yt-dlp` に未保存なら、既存の `yt-dlp` skillのスクリプトで自動保存される。
3. `scan_ui_*.jpg` で動画全体のカードUIを流し見し、`candidate_*_full.jpg` と `candidate_*_ui.jpg` で候補区間を詳しく見る。
   - 1つの候補画像に複数のEX実行が入ることがある。特に連続発動では候補内の全フレームを読む。
   - 黄色の数値はそのシート/候補内の相対秒であり、TLに出す時刻ではない。
   - コスト減少があるのに選択カード状態が見えない候補は、誤検出として捨てず、30fps以上の高密度クロップを作ってカード消失・スキル名バナー・演出を確認する。
4. 各EX実行について、次を確認する。
   - 実行直前の最後のグレーアウト/選択カード状態
   - 直後にコストが減っていること
   - 実行カードが消えた、または次カードに入れ替わったこと
   - 高速タップや即時発動では、カードが浮く/キャンセルが出るフレームが動画上に残らないことがある。この場合でも、コスト減少、カード消失または置換、直後のスキル名バナー/固有演出がそろえば有料EXとして記録する。
5. 実行直前の最後のグレーアウトフレームから戦闘タイマーを読み取る。右上の `03:43.500` のようなゲーム内タイマーを使い、動画再生時刻は出力しない。
   - 選択カード状態が見えない高速発動では、コスト減少直前または直後のスキル名バナー開始フレームから戦闘タイマーを読み、候補区間内で最も実行瞬間に近い時刻を使う。
6. 同じフレームのコストバーから実行時コストを読む。最大コストはバーの箱数で判定する。10.5なら右端の箱が半分、11なら11箱。
7. 選択カードの生徒名を読む。動画だけで曖昧なら、Wikiruのキャラクター一覧を開いてカード顔・髪型・衣装・属性色を照合する。スキル効果テキストだけで生徒名を決めない。

Wikiru参照:

```text
https://bluearchive.wikiru.jp/?キャラクター一覧
```

## コストの読み方

- 青い箱の満タン数を整数部にする。
- 次の箱が途中まで青い場合は、その充填率を小数1桁に丸める。
- 例: 7箱満タンで8箱目が約70%なら `7.7`。
- コスト円の整数表示だけに頼らない。小数部はバーの塗りから読む。
- 青いコストが1以上ある状態で発動し、その直後にオレンジ/マイナス表示などの特殊状態へ入るEXは有料発動として記録する。
- 特殊状態に入った後の0コスト追撃カードや、コスト減少を伴わない発動は初版では除外する。

## カード名の照合

- 生徒名の根拠の優先順位は、選択カードのポートレート/衣装、表示されたスキル名、Wikiru個別ページのEX名とコスト、効果テキストの順にする。
- 画面上のカード名が見えない場合は、選択カードのポートレートをWikiruのキャラクター一覧と照合する。効果テキストは補助情報であり、似た効果の別生徒へ誤って飛ばないようにする。
- 衣装違いは `ハレ(キャンプ)`、`ヒナ(ドレス)` のように動画/編成/カード絵に合う名前で書く。
- コラボ生徒も一覧または個別ページで確認する。表記は日本語版の一般的な表記を使う。
- 例: `自身を除く味方1人に対して、会心ダメージ率を83.4%増加(30秒間)` は `アル(ドレス)` の `興味深い提案`。会心ダメージ率バフだからといって `アコ` と推定しない。

## 高速発動・見落とし対策

- カードが浮く、画面がグレーアウトする、`キャンセル` が出る、という状態は強い証拠だが必須条件ではない。
- 高速タップ、対象指定直後の即発動、またはフレーム落ち/低fpsサンプルでは、選択状態が候補画像に1枚も写らないことがある。
- その場合は、次の3点でEX実行を判定する。
  - 直前の手札にあるカードのコストと、直後のコスト減少幅が一致または近い。
  - そのカードが消える、コスト表示が変わる、または次カードへ置き換わる。
  - 直後に該当生徒のスキル名バナー、固有演出、または攻撃/バフ効果が始まる。
- 特に `ヒナ(ドレス)` のように発動後の専用状態や追加入力が絡むEXは、手札カードの見た目がすぐ変化する。通常のカード浮き上がりだけを待つと見落とすため、コスト減少とカード置換を必ず見る。
- 高速発動を疑う区間では、`ffmpeg` で30fpsから60fps程度のUIクロップとフルフレームクロップを作り、コスト減少の前後0.5秒以上を確認する。

## 出力

時系列順に、確信できた有料EX実行だけを出力する。

```text
7.7 (3:43.500) カヨコ
3.1 (3:42.633) クルミ
3.3 (3:37.433) ナギサ
```

判断が曖昧な行は最後に短い注記を付ける。自信が低いカード名は無理に確定せず、候補名と確認に使った画像ファイルを示す。
